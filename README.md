# 销售情报助手 · Sales Intelligence Skill

> 最后更新：2026-03-19

## 产品定位

**OpenClaw Skill，搭载在"小龙虾"Agent 上，通过飞书推送日报。**

核心价值：以分众销售视角，每天自动搜索品牌/行业动态，识别广告投放商机，输出可直接行动的销售情报。

核心问题：**这个品牌接下来有没有可能需要投广告？**

---

## 架构设计

### Skill 与小龙虾的分工

- **Skill 负责干活**：搜索、分析、生成日报、响应查询
- **小龙虾负责调度**：记忆用户偏好，维护品牌配置，在合适时机调用 Skill
- 客户配置存在 Skill 的 `config.yaml` 中，由小龙虾通过对话维护

### 搜索分层策略

| 层级 | 频率 | 搜索范围 | 搜索深度 | 成本 |
|------|------|---------|---------|------|
| 高频窄搜 | 每天 8:00 | 品牌名 + 高价值关键词 | basic (1 credit) | ~30 credits/天 |
| 低频宽搜 | 每周一、四 | 行业 + 政策 + 竞品 | advanced (2 credits) | ~20 credits/周 |
| 事件驱动 | 用户触发 | Deep Research | advanced | 按需 |

### 高价值信号（从高到低）

1. 融资/投资 → 有预算了，要打市场
2. 新品即将发布 → 需要上市推广
3. 高管/CMO 变动 → 新官上任要花钱
4. 进入新市场/新品类 → 需要建立认知
5. 竞品在猛投广告 → 压力之下可能跟进
6. 行业政策利好（补贴、新标准）→ 整个行业加大投放
7. 已有广告投放 → 后续可以争取

## 技术方案

### 信息采集

- **主搜索引擎**：Tavily API（`topic: "news"` + `time_range: "day"`）
- **英文搜索**：Tavily（Insta360 等有海外业务的品牌）
- **去重机制**：本地维护已推送 URL 列表，新结果先比对
- **未来扩展**：新榜 API（公众号）、IT桔子 API（融资）

### AI 分析

- 使用 OpenClaw 自带模型能力（Kimi 2.5）
- 每条信息判断：相关性、紧迫度、商机切入点
- 过滤噪音：纯技术内容、旧闻、SEO 垃圾

### 输出

- **格式**：Markdown 文本
- **渠道**：飞书（通过 OpenClaw Channel）
- **频率**：每天 8:00，无新信息则不推送
- **结构**：按品牌分组，每条附原文链接
