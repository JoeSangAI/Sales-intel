## 二、根因详解

### 根因 A：distribute_results() 模糊匹配导致跨赛道分发

**文件**: `scripts/search_pool.py` 第 213-228 行

```python
if track in profile_tracks or any(t in track or track in t for t in profile_tracks):
```

皮俊坤的赛道配置：
- 运动户外：`跑步健身 品牌 营销 广告` → 匹配到世运电路（某条 PCB 新闻含"跑步"字样）
- 医药大健康：`大健康 品牌 营销` → 匹配到千寻智能（具身智能公司新闻含"健康"字样）

**搜索词 → 赛道名的映射链条没有语义校验**，只要关键词命中就分发。

---

### 根因 B：缓存无日期校验，历史数据跨日扩散

**文件**: `scripts/search_pool.py` 第 104-111 行（缓存加载）、第 137 行（time_range 固定为 day）

**问题链条**:
1. 周一行业搜索结果存入缓存（含融资结果）
2. 周二 `include_industry=False`，融资搜索不执行
3. 但 `execute_shared_search()` 加载缓存时没有校验日期
4. `distribute_results()` 无条件分发缓存中所有结果
5. 所有档案收到跨时间的融资数据

**证据**: 皮俊坤周二收到 171 条分发结果，但周二是禁止行业融资搜索的。

---

### 根因 C：_NOISE_DOMAINS 不完整

**文件**: `scripts/search.py` 第 265-272 行

现有列表缺少：
- 马拉松/体育赛事报道（如 marathon.、race. 等域名模式）
- 港股投资日志网站（如 zutong.cn、ztock.cn）

---

### 根因 D：千问品牌 sub_brands 歧义

**文件**: `config.yaml` 第 43-48 行

```yaml
- name: 千问
  sub_brands: [通义千问, 千问, Qwen, 淘宝闪购, 阿里云]
```

搜索"千问"时 Bocha 可能扩展到"千寻智能"（Qianxun AI，具身智能公司），导致跨赛道结果。

---

### 根因 E：消费品赛道关键词过少

**文件**: `scripts/search.py` 第 498 行

```python
"消费品": ["新消费融资", "消费品牌融资", "新品牌融资", "消费独角兽", "快消品融资"],
```

AI大模型有 7 个精准词，消费品只有 5 个宽泛词，覆盖不足。

### 根因 F：融资去重未跨事件

**文件**: `scripts/dedup.py` `dedup_fundraising_by_company()`

公司名归一化去重保留每公司一条，但**同公司不同融资事件**视为不同条目。金凯撒两条不同事件的融资新闻都进了报告。

## 三、修改方案

### 方案 1（P0）：缓存加载增加日期校验

**文件**: `scripts/search_pool.py` 第 104-111 行

**现状**: 缓存命中就直接用，不检查日期

**修改**: 缓存必须是当天才用

```python
# 第 104-111 行，修改为：
if os.path.exists(cache_path):
    try:
        with open(cache_path, "r", encoding="utf-8") as f:
            cached = json.load(f)
        cache_date = cached.get("date", "")
        today = datetime.now().strftime("%Y-%m-%d")
        if cache_date == today:
            print(f"  [搜索池] 加载缓存（同一天）: {len(cached.get('results', {}))} 条查询")
            return cached.get("results", {})
        else:
            print(f"  [搜索池] 缓存过期（缓存 {cache_date}，今天 {today}），忽略")
    except Exception as e:
        print(f"  [警告] 加载搜索池缓存失败: {e}")
```

## 四、质检层改造（两层架构）

### 现状问题

`quality_check.py` 完全依赖 LLM 判断：
- 结果不稳定（同样输入可能不同判断）
- 无明确 checklist
- 曹茸连续幻觉失败后仍生成报告（只是标记人工审核）

### 改造方案：新增规则校验层

#### 新增文件: `scripts/quality_rules.py`

```python
"""
质检第一层：规则校验（确定性，零 LLM 调用）
"""

import re
from datetime import datetime


_NOISE_DOMAINS_QC = [
    "chinairn.com", "chinabgao.com", "askci.com",
    "stockstar.com", "stock.sohu.com",
    "trustexporter.com", "globalimporter.net",
    "topnews.cn", "bbs.q.sina.com.cn", "winshang.com",
    "zutong.cn", "ztock.cn", "stockway.cn",
]


def _is_noise_domain(url: str) -> bool:
    from urllib.parse import urlparse
    domain = urlparse(url).netloc.lower()
    return any(d in domain for d in _NOISE_DOMAINS_QC)


def run_rules_check(report: str, original_items: list, profile: dict) -> dict:
    """
    第一层规则校验。

    返回:
        {
            "pass": bool,        # True = 全部通过
            "issues": list,      # 致命问题（阻断）
            "warnings": list,    # 非致命警告
        }
    """
    issues = []
    warnings = []

    # ── 构建白名单集合 ──
    brand_names = set()
    for b in profile.get("brands", []):
        brand_names.add(b.get("name", "").lower())
        for sub in b.get("sub_brands", []):
            brand_names.add(sub.lower())

    track_names = {t.get("name", "") for t in profile.get("fundraising", {}).get("tracks", [])}
    industry_names = {i.get("name", "") for i in profile.get("industries", [])}
    allowed_sectors = track_names | industry_names

    # ── 提取报告中的所有 URL ──
    report_urls = re.findall(r'\[([^\]]+)\]\((https?://[^\)]+)\)', report)

    # ── 构建原始数据 URL 集合 ──
    valid_urls = {item.get("url", "") for item in original_items}

    # ── R1: URL 必须在原始数据中 ──
    for text, url in report_urls:
        if url not in valid_urls:
            issues.append(f"[R1 阻断] 链接不在原始数据中: {url}")

    # ── R3: 日期真实性（超过7天警告） ──
    date_pattern = re.compile(r'20\d{2}[-/]\d{2}[-/]\d{2}')
    for text, url in report_urls:
        dates_in_report = date_pattern.findall(text)
        for d in dates_in_report:
            try:
                date_str = d.replace('/', '-')
                pub = datetime.strptime(date_str, "%Y-%m-%d")
                diff = (datetime.now() - pub).days
                if diff > 7:
                    warnings.append(f"[R3 警告] 日期过早（{diff}天前）: {d}，链接: {url}")
            except:
                pass

    # ── R5: 噪音域名 ──
    for text, url in report_urls:
        if _is_noise_domain(url):
            issues.append(f"[R5 阻断] 噪音域名: {url}")

    return {
        "pass": len(issues) == 0,
        "issues": issues,
        "warnings": warnings,
    }
```

#### 修改文件: `scripts/quality_check.py`

在 `quality_check()` 函数开头插入规则校验：

```python

        rules_result = run_rules_check(report, original_items, profile)
        if not rules_result["pass"]:
            print(f"  [QC 规则失败] 阻断 {len(rules_result['issues'])} 个")
            for issue in rules_result["issues"]:
                print(f"    - {issue}")
            return {
                "format_ok": False,

                "rules_issues": rules_result["issues"],
                "rules_warnings": rules_result["warnings"],

            print(f"  [QC 规则警告] {len(rules_result['warnings'])} 个（非阻断）")
```
